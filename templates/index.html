How does Hybrid Search work?

This app uses MongoDB Atlas Search's $rankFusion to combine two powerful search methods, giving you the most relevant results every time. Think of it like having two different detectives working on your case.

1. Full-Text Search: The "Keyword Detective"

This is your traditional search. It's like using a book's index to find an exact word. It excels at finding literal matches for variables like user_id or functions like connectToDB().

2. Vector Search: The "Concept Detective"

This is where the magic happens. Instead of matching words, this search understands the meaning and intent behind your code by converting it into "embeddings."

What are Text Embeddings?

An embedding is a numerical fingerprint that represents the semantic meaning of a piece of code. Snippets with similar purposes will have mathematically similar fingerprints, even if they don't share the same keywords. For example, these three functions all describe the same concept and would have very close embeddings:

def authenticate_user_credentials(): const handleLogin = () => {} func (s *AuthService) VerifyPassword() {}

When you search for "a function to verify users," your query is also converted into an embedding. Vector search then finds the code whose fingerprint is the closest match. It's searching by vibe and context, not just keywords.

Blending the Results with $rankFusion

$rankFusion acts as the master detective, taking the reports from both detectives and producing a single, intelligently ranked list. It gives the highest scores to results that are both conceptually relevant and contain your precise keywords, giving you the best of both worlds.
